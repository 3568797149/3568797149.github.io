关于博客搭建中报错问题的解决
搭建博客之路因为报错而艰难，所以写的第一篇学习文章是关于博客搭建过程中自己遇到的报错以及问题的解决办法

初次建立git仓库

★
指令：git clone git@github.com:3568797149/test.git
报错：
Cloning into ‘test’…
warning: You appear to have cloned an empty repository
问题分析：

  • 确认仓库非空: 登录到 GitHub，访问你的仓库，检查是否有文件或文件夹存在。

  • 添加文件并提交： 如果仓库确实是空的，你可以通过以下步骤添加文件：
    ‘cd test’ # 进入克隆的仓库目录
    ‘touch README.md’ # 创建一个 README 文件，或者添加其他文件
    ‘git add’ # 将文件添加到暂存区
    ‘git commit -m “Initial commit” ‘ # 提交更改
    ‘git push origin master’ # 推送到 GitHub 仓库
    ps:在建立新仓库一定要勾选如下选项才会自动生成README.md文件！！！

★
指令：git init
报错：fatal: unable to access ‘https://github.com/hexojs/hexo-starter.git/‘: Recv failure: Connection was reset
问题分析：此问题有可能因为网络连接问题导致（个人认为概率很小），还有可能就是绑定GitHub时站点配置文件中repo应设置为SSH公钥而不是HTTP（前提是获取公钥时指令输入的是获取SSH）

node.js和hexo命令的使用

★
指令： hexo init
报错node:internal/modules/cjs/loader:1080 throw err; ^ Error: Cannot find module ‘D:\Node\node_global\node_modules\hexo-cli\bin\hexo’ at Module._r
问题分析：这个错误表明 Node.js 在加载 Hexo 模块时遇到了问题。
问题解决：

  • 重新安装 Hexo-cli：
    执行以下命令可以重新安装Hexo-cli：

npm uninstall hexo-cli -g
npm install hexo-cli -g
这将卸载并重新安装 Hexo-cli，确保它的路径正确。

  • 检查 Node.js 环境变量：
    确保你的 Node.js 全局模块的路径（D:\Node\node_global）被正确添加到系统的 PATH 环境变量中。如果没有添加，可以手动添加它。

★
指令：hexo d
报错：spwan failed
问题分析：在将Hexo部署到GitHub过程中repo没有正确输入，应输入SSH格式的公钥而不是HTTP
